# Volundr - Terraform Module

## Overview

The Terraform module generates production-ready Terraform modules with multi-cloud support, proper variable definitions, outputs, versioning, and documentation.

## Models

### CloudProvider

Supported cloud providers:
- `AWS` - Amazon Web Services
- `AZURE` - Microsoft Azure
- `GCP` - Google Cloud Platform
- `GENERIC` - Provider-agnostic modules

### ResourceCategory

Resource categories:
- `COMPUTE` - VMs, containers, serverless
- `NETWORKING` - VPC, subnets, load balancers
- `STORAGE` - Object storage, block storage, file systems
- `DATABASE` - RDS, Cloud SQL, Cosmos DB
- `SECURITY` - IAM, KMS, secrets
- `MONITORING` - CloudWatch, Stackdriver, Azure Monitor

### ModuleComplexity

Module complexity levels:
- `SIMPLE` - Single resource, minimal configuration
- `STANDARD` - Multiple resources, common patterns
- `ADVANCED` - Complex dependencies, multi-region

### ModuleConfig

Main configuration model:

```python
from Volundr.Terraform import (
    ModuleConfig, ModuleBuilder, CloudProvider,
    ResourceCategory, ModuleComplexity, VariableConfig, OutputConfig
)

config = ModuleConfig(
    name="vpc",
    provider=CloudProvider.AWS,
    category=ResourceCategory.NETWORKING,
    complexity=ModuleComplexity.STANDARD,

    description="Production VPC with public and private subnets",

    # Variables
    variables=[
        VariableConfig(
            name="vpc_cidr",
            type="string",
            description="CIDR block for the VPC",
            default="10.0.0.0/16",
            validation={"condition": "can(cidrnetmask(var.vpc_cidr))"}
        ),
        VariableConfig(
            name="environment",
            type="string",
            description="Environment name",
            validation={"condition": "contains([\"dev\",\"staging\",\"prod\"], var.environment)"}
        ),
        VariableConfig(
            name="enable_nat",
            type="bool",
            description="Enable NAT Gateway",
            default=True
        )
    ],

    # Outputs
    outputs=[
        OutputConfig(name="vpc_id", description="VPC ID", value="aws_vpc.main.id"),
        OutputConfig(name="public_subnets", description="Public subnet IDs", value="aws_subnet.public[*].id"),
        OutputConfig(name="private_subnets", description="Private subnet IDs", value="aws_subnet.private[*].id")
    ],

    # Tags
    tags={"ManagedBy": "Terraform", "Generator": "Volundr"},

    # Features
    include_examples=True,
    include_tests=True,
    terraform_version=">=1.5.0"
)
```

## Service: ModuleBuilder

### Basic Usage

```python
from Volundr.Terraform import ModuleConfig, ModuleBuilder, CloudProvider, ResourceCategory

# Create configuration
config = ModuleConfig(
    name="vpc",
    provider=CloudProvider.AWS,
    category=ResourceCategory.NETWORKING,
    description="Production VPC module"
)

# Generate module
builder = ModuleBuilder()
module = builder.generate(config)

# Access generated files
print(module.main_tf)        # Main resource definitions
print(module.variables_tf)   # Variable definitions
print(module.outputs_tf)     # Output definitions
print(module.versions_tf)    # Provider versions
print(module.locals_tf)      # Local values
print(module.readme_md)      # Documentation

# Optional files
print(module.examples)       # Example configurations
print(module.tests)          # Terratest files

# Check quality
print(f"Best Practice Score: {module.best_practice_score}/100")

# Save to directory
builder.save_to_directory(module, output_dir="./terraform/modules/vpc")
```

### Generated Files

The generator produces a complete Terraform module:

```
modules/vpc/
├── main.tf           # Resource definitions
├── variables.tf      # Input variables
├── outputs.tf        # Output values
├── versions.tf       # Provider requirements
├── locals.tf         # Local values
├── README.md         # Auto-generated documentation
├── examples/
│   └── basic/
│       ├── main.tf
│       └── outputs.tf
└── tests/
    └── vpc_test.go   # Terratest file
```

### Example Output

**main.tf**
```hcl
# VPC Module - Generated by Volundr
# Provider: AWS
# Category: Networking

resource "aws_vpc" "main" {
  cidr_block           = var.vpc_cidr
  enable_dns_hostnames = true
  enable_dns_support   = true

  tags = merge(local.common_tags, {
    Name = "${var.name}-vpc"
  })
}

resource "aws_internet_gateway" "main" {
  vpc_id = aws_vpc.main.id

  tags = merge(local.common_tags, {
    Name = "${var.name}-igw"
  })
}

resource "aws_subnet" "public" {
  count = length(var.public_subnet_cidrs)

  vpc_id                  = aws_vpc.main.id
  cidr_block              = var.public_subnet_cidrs[count.index]
  availability_zone       = var.availability_zones[count.index]
  map_public_ip_on_launch = true

  tags = merge(local.common_tags, {
    Name = "${var.name}-public-${count.index + 1}"
    Tier = "public"
  })
}
```

**variables.tf**
```hcl
variable "name" {
  type        = string
  description = "Name prefix for all resources"
}

variable "vpc_cidr" {
  type        = string
  description = "CIDR block for the VPC"
  default     = "10.0.0.0/16"

  validation {
    condition     = can(cidrnetmask(var.vpc_cidr))
    error_message = "Must be a valid CIDR block."
  }
}

variable "environment" {
  type        = string
  description = "Environment name"

  validation {
    condition     = contains(["dev", "staging", "prod"], var.environment)
    error_message = "Environment must be dev, staging, or prod."
  }
}

variable "enable_nat" {
  type        = bool
  description = "Enable NAT Gateway for private subnets"
  default     = true
}
```

**outputs.tf**
```hcl
output "vpc_id" {
  description = "VPC ID"
  value       = aws_vpc.main.id
}

output "public_subnets" {
  description = "Public subnet IDs"
  value       = aws_subnet.public[*].id
}

output "private_subnets" {
  description = "Private subnet IDs"
  value       = aws_subnet.private[*].id
}
```

**versions.tf**
```hcl
terraform {
  required_version = ">=1.5.0"

  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}
```

## Best Practice Score

The builder calculates a score (0-100) based on:

| Criteria | Weight | Description |
|----------|--------|-------------|
| Variables | 20 | Input variables with descriptions and validation |
| Outputs | 15 | Outputs with descriptions |
| Locals | 10 | Local values for DRY code |
| Versioning | 15 | Provider version constraints |
| Documentation | 20 | README with examples |
| Examples | 10 | Example configurations |
| Tests | 10 | Terratest files |

## CLI Usage

```bash
# Basic module generation
python -m Volundr tf generate --name vpc --provider aws --category networking

# With options
python -m Volundr terraform generate \
  --name vpc \
  --provider aws \
  --category networking \
  --description "Production VPC module" \
  --complexity standard \
  --output ./terraform/modules
```
